Erreur au démarrage de GDM

Cette section s'applique si vous voyez le message suivant au démarrage de Tails :

Error starting GDM with your graphics card: nom de votre carte graphique [identifiant] (numéro de version). Please take note of this error and visit https://tails.boum.org/gdm for troubleshooting.

  1. Identifiez le nom, l'identifiant et le numéro de version (si il existe) de votre carte graphique.

    Par exemple, si votre message d'erreur commence par :

    Error starting GDM with your graphics card: NVIDIA Corporation GT218M [NVS 3100M] [10de:0a6c] (rev a2)

    • Le nom est NVIDIA Corporation GT218M [NVS 3100M].
    • L'identifiant est [10de:0a6c]. L'identifiant est unique à votre modèle de carte graphique, il n'est pas unique à votre ordinateur.
    • Le numéro de version est a2. Votre carte graphique peut ne pas avoir de numéro de version.

  2. Vérifiez si votre carte graphique est listée ci-dessous. Par exemple, vous pouvez chercher son nom ou son identifiant dans cette page.

    • Si votre carte graphique est listée, vérifiez si un contournement est documenté pour la faire marcher dans Tails.

      Si le contournement ne marche pas, veuillez contacter notre équipe d'assistance par email.

      Mentionnez dans votre email:

      • La version de Tails que vous essayez de démarrer.
      • Le nom, l'identifiant et le numéro de version (si il existe) de votre carte graphique.
      • Le contournement que vous avez essayé et qui a échoué.

    • Si votre carte graphique n'est pas listée, veuillez contacter notre équipe d'assistance par email.

      Mentionnez dans votre email:

      • La version de Tails que vous essayez de démarrer.
      • Le nom, l'identifiant et le numéro de version (si il existe) de votre carte graphique.

      Vous pouvez nous envoyer une photo du message d'erreur tel qu'il apparaît sur votre écran.

  3. Si votre problème est corrigé dans une future version de Tails, veillez nous le faire savoir pour que nous puissions mettre à jour cette page.

Autres problèmes

Écran noir avec des ordinateurs avec cartes graphiques commutables

Certains ordinateurs avec cartes graphiques commutables (comme Optimus) échouent dans le choix de la carte vidéo et se bloquent sur un écran noir. Ceci a été constaté sur MacBook Pro 6.2, MacBook Pro 10.1 Retina, MacBook Pro 15 pouces (début 2011) et peut en affecter beaucoup d'autres.

Il y a plusieurs possibilités pour contourner ce problème :

  • Sélectionnez explicitement une des deux cartes graphiques dans le BIOS au lieu de laisser le système en choisir une automatiquement. Si cela ne résout pas le problème, essayez en sélectionnant l'autre carte graphique.

  • Pour les ordinateurs Mac, il est possible d'utiliser une application tierce, http://gfx.io/, pour forcer la carte graphique intégrée uniquement avec macOS. Redémarrez dans ce mode spécial pour faire fonctionner Tails.

  • Les personnes expertes de Linux peuvent aussi :

    1. Ajouter l'option i915.modeset=0 rootpw=pass dans le menu du chargeur d’amorçage.

    2. Créer un fichier /etc/X11/xorg.conf.d/switchable.conf avec le contenu suivant :

      Section "Device"
    Identifier "Device0"
    Driver "nouveau"
    BusID "1:0:0"
EndSection

    3. Redémarrer X avec la commande :

      service gdm3 restart

    4. Après le démarrage de la session GNOME, modifier à nouveau le mot de passe root avec la commande :

      sudo passwd

Pour plus de détails, voir notre ticket Video is broken with switchable graphics.

La session GNOME ne peut pas démarrer avec des ordinateurs avec cartes graphiques commutables

Sur certains ordinateurs avec cartes graphiques commutables, Tails 2.10 et suivants ne peuvent ouvrir une session GNOME et reviennent systématiquement au Tails Greeter.

Démarrer en mode Troubleshooting Mode marche, ainsi que le fait d'ajouter modeprobe.blacklist=nouveau dans les options de démarrage.

Machines virtuelles avec VMware

Pour améliorer le support de Tails tournant à l'intérieur d'une machine virtuelle avec VMware, installez le paquet open-vm-tools-desktop dans Tails.
