Faire un don

Ces instructions permettent de copier manuellement vos données persistantes vers une nouvelle clé USB. Suivez-les si vous avez de bonnes raisons de penser que vos paramètres de persistance sont corrompus ou si vous souhaitez prendre une précaution supplémentaire.

Créer une nouvelle clé USB

  1. Installez la dernière version de Tails sur une nouvelle clé USB en suivant les instructions d'installation. N'utilisez pas de clé USB Tails potentiellement corrompue pour créer cette nouvelle clé USB.

  2. Créez un volume persistant sur cette nouvelle clé USB. Nous vous conseillons d'utiliser une phrase de passe différente pour protéger le nouveau volume persistant.

  3. Activez sur la nouvelle clé USB les options de persistance que vous souhaitez utiliser.

  4. Redémarrez sur la nouvelle clé USB, activez la persistance et définissez un mot de passe d'administration.

Récupérer vos données depuis l'ancienne clé USB Tails

Monter l'ancien volume persistant

  1. Branchez l'ancienne clé USB depuis laquelle vous souhaitez récupérer vos données.

  2. Choisissez Applications ▸ Utilitaires ▸ Disques pour ouvrir l'utilitaire Disques de GNOME.

  3. Dans le panneau de gauche, cliquez sur la clé USB correspondant à l'ancienne clé USB Tails.

  4. Dans le panneau de droite, cliquez sur la partition marquée comme LUKS. Le nom de cette partition doit être TailsData.

  5. Cliquez sur le bouton pour déverrouiller l'ancien volume persistant. Saisissez la phrase de passe de l'ancien volume persistant puis cliquez sur Déverrouiller.

  6. Cliquez sur la partition TailsData qui est apparue sous la partition intitulée LUKS.

  7. Cliquez sur le bouton . L'ancien volume persistant est maintenant monté sur /media/amnesia/TailsData.

Copier vos anciens fichiers vers le nouveau volume persistant

  1. Choisissez Applications ▸ Outils système ▸ Terminal administrateur pour ouvrir un terminal avec les droits d'administration.

  2. Exécuter la commande nautilus pour ouvrir le navigateur de fichier avec les droits d'administration.

  3. Dans le panneau de gauche, cliquez sur Autres emplacements.

  4. Dans le panneau de droite, naviguez jusqu'à Ordinateur ▸ media ▸ amnesia ▸ TailsData pour ouvrir l'ancien volume persistant.

  5. Dans la barre de titre, choisissez  ▸ pour ouvrir un nouvel onglet.

  6. Dans le panneau de gauche, cliquez sur Autres emplacements.

  7. Dans le panneau de droite, naviguez jusqu'à Ordinateur ▸ live ▸ persistence ▸ TailsData_unlocked pour ouvrir le nouveau volume persistant.

  8. Cliquez sur l'onglet TailsData.

  9. Pour copier un dossier contenant des données persistantes depuis l'ancien volume persistant vers le nouveau, faites glisser puis déposez ce dossier depuis l'onglet TailsData vers l'onglet TailsData_unlocked.

    Lors de la copie d'un dossier, sélectionnez l'option Appliquer cette action pour tous les fichiers et cliquez sur Fusionner pour l'appliquer à tous les sous-dossiers. Puis vous avez à sélectionner à nouveau l'option Appliquer cette action pour tous les fichiers et cliquez sur Remplacer pour l'appliquer à tous les fichiers.

    Ne copiez pas un dossier si vous ne savez pas à quoi il sert.

    • Le dossier apt et le fichier live-additional-software.conf correspondent aux options de persistance des [[Logiciels additionnels|configure#additional_software]. Mais il est nécessaire d'avoir les droits d'administration pour les importer, et cela sort du domaine couvert par ces instructions. Notez que ce dossier ne contient pas de données personnelles.
    • Le dossier bookmarks correspond à l'option de persistance Marque-pages du navigateur.
    • Le dossier cups-configuration correspond à l'option de persistance Imprimantes.
    • Le dossier dotfiles correspond à l'option de persistance Dotfiles.
    • Le dossier electrum correspond à l'option de persistance Client Bitcoin.
    • Le dossier gnupg correspond à l'option de persistance GnuPG.
    • Le dossier thunderbird correspond à l'option de persistance Thunderbird.
    • Le dossier nm-connections correspond à l'option de persistance Connexions Réseaux.
    • Le dossier openssh-client correspond à l'option de persistance Client SSH.
    • Le dossier Persistent correspond à l'option de persistance Données personnelles.
    • Le dossier pidgin correspond à l'option de persistance Pidgin.

  10. Après la copie, fermez le navigateur de fichiers.

  11. Dans le terminal, exécutez la commande suivante pour corriger la propriété de vos fichiers personnels :

   find /live/persistence/TailsData_unlocked/ -uid 1000 -exec chown -R 1000:1000 '{}' \;
