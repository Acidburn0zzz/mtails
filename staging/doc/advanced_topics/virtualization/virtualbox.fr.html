Il est possible d'utiliser Tails dans une machine virtuelle avec VirtualBox depuis un système d'exploitation Windows, Linux, ou macOS..

Utiliser Tails dans une machine virtuelle a de nombreuses conséquences en terme de sécurité. En fonction du système d’exploitation hôte et de vos besoins en terme de sécurité, faire tourner Tails dans une machine virtuelle peut être dangereux.

VirtualBox a une version libre, appelée VirtualBox Open Source Edition et des composants propriétaires, par exemple pour ajouter la prise en charge des périphériques USB.

Pour des raisons de sécurité, nous recommandons d'utiliser uniquement la version Open Source Edition, même si elle ne permet pas d'utiliser de volume persistant.

Avec la fonctionnalité de dossiers partagés de VirtualBox vous pouvez avoir accès à des fichiers de votre système hôte depuis votre système invité.

Assurez-vous de comprendre les implications en terme de sécurité de l'accès aux disques durs internes depuis Tails avant d'utiliser cette fonctionnalité.

Considérations de sécurité pour Windows et macOS

Dans nos avertissements de sécurité à propos de la virtualisation nous recommandons d'utiliser Tails dans une machine virtuelle seulement si le système d'exploitation est de confiance.

Microsoft Windows et macOS étant des logiciels propriétaires, ils ne peuvent pas être considérés de confiance. Utilisez Tails dans une machine virtuelle hébergée par Windows ou macOS seulement pour faire des tests et ne comptez pas dessus pour de la sécurité.

Installation

Pour installer VirtualBox dans Debian ou Ubuntu, exécutez la commande suivante :

sudo apt install virtualbox

Pour des instructions d'installation de VirtualBox pour d'autres systèmes d'exploitations, consultez la documentation de VirtualBox .

Utiliser Tails depuis une image ISO

Tout d'abord, lancer VirtualBox.

Pour créer une nouvelle machine virtuelle :

  1. Choisir Machine ▸ Nouvelle....
  2. Dans la fenêtre Nom et système d'exploitation, mettre :
    • Un nom de votre choix.
    • Type : Linux.
    • Version : Autre Linux (64 bit).
    • Cliquer sur Suivant.
  3. Dans la fenêtre Taille de la mémoire :
    • Allouer au moins 2048 Mo de RAM.
    • Cliquer sur Suivant.
  4. Dans la fenêtre Disque dur :
    • Choisir Ne pas ajouter de disque dur virtuel.
    • Cliquer sur Créer.
    • Cliquer sur Continuer dans la fenêtre d'avertissement à propos de la création de machine virtuelle sans disque dur.

Pour configurer la machine virtuelle pour démarrer depuis une image ISO :

  1. Sélectionner la machine virtuelle dans la partie gauche.
  2. Choisir Machine ▸ Configuration....
  3. Sélectionner Système dans la partie gauche.
  4. Dans la section Fonctions avancées de l'onglet Carte mère, vérifier que la case Activer les IO-APIC est cochée.
  5. Sélectionner Stockage dans la partie gauche.
  6. Sélectionner Vide sous Contrôleur IDE dans la liste de sélection de l'Arborescence de stockage dans la partie droite.
  7. Cliquer sur l'icône CD dans la droite de la fenêtre et sélectionner Choisissez un fichier de CD/DVD virtuel... pour naviguer jusqu'à l'image ISO de Tails.
  8. Cocher l'option Live CD/DVD.
  9. Cliquer sur OK.

Pour démarrer la nouvelle machine virtuelle :

  1. Sélectionner la machine virtuelle dans la partie gauche de la fenêtre.
  2. Cliquer sur Démarrer.
